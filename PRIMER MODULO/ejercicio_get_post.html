Ejercicio en clase:
GET y POST en acción

Aprende cómo viajan los datos entre el navegador y el servidor.

Objetivos del ejercicio

🔍

Construir un formulario con método GET y ver los datos en la URL.

🔐

Construir un formulario con método POST para datos sensibles.

🔬

Inspeccionar el tráfico HTTP con las DevTools del navegador.

🧠

Decidir cuándo usar cada método en situaciones reales.

Contexto del escenario

📋 Situación

Eres el desarrollador frontend de "TiendaYa", una tienda en línea. Tu jefe te pide construir dos páginas: un buscador de productos (donde el usuario filtra por nombre y categoría) y un formulario de inicio de sesión (donde ingresa correo y contraseña). Debes elegir el método HTTP correcto para cada uno.

Parte 1 · Formulario de búsqueda con GET

1A
Crea el archivo buscar.html
GET

Crea un formulario que envíe una búsqueda. El formulario debe enviar los datos usando método GET a la acción # (por ahora, sin servidor real).

  • El formulario tiene el atributo method="GET"
  • Campo de texto: "Nombre del producto" → name="producto"
  • Campo de texto: "Categoría" → name="categoria"
  • Un botón de tipo submit con el texto "Buscar"
  • Cada campo tiene su respectivo <label>
<!-- buscar.html · Completa los blancos con __ --> <form action="#" method="______"> <label for="producto">Nombre del producto:</label> <input type="______" id="producto" name="______" placeholder="ej: zapatos"> <label for="categoria">Categoría:</label> <input type="text" id="______" name="categoria"> <button type="______">______</button> </form>
1B
Observa la URL en el navegador
GET

Abre buscar.html en el navegador. Escribe "zapatos" en producto y "ropa" en categoría. Haz clic en Buscar.

  • Copia aquí la URL que aparece en la barra de dirección:
URL: _______________________________________________________________________
  • ¿Puedes ver los datos que escribiste en la URL? (Sí / No): _______
  • ¿Cuál es el símbolo que separa la URL base de los parámetros?: _______
  • ¿Cuál es el símbolo que separa los parámetros entre sí?: _______
💡 Pista

Observa bien la URL. Debería verse algo así: buscar.html?producto=zapatos&categoria=ropa

Parte 2 · Formulario de login con POST

2A
Crea el archivo login.html
POST

Ahora construye el formulario de inicio de sesión. Los datos no deben aparecer en la URL. Usa el método POST.

  • El formulario tiene el atributo method="POST"
  • Campo de tipo email: "Correo electrónico" → name="correo"
  • Campo de tipo password: "Contraseña" → name="clave"
  • Botón de envío con el texto "Iniciar sesión"
  • Bonus: Agrega un campo checkbox "Recordarme" → name="recordar"
<!-- login.html · Completa el formulario desde cero --> <!DOCTYPE html> <html lang="es"> <head> <meta charset="UTF-8"> <title>Login - TiendaYa</title> </head> <body> <h1>Iniciar sesión</h1> <!-- Escribe el formulario aquí --> ___________________________________________________ ___________________________________________________ ___________________________________________________ ___________________________________________________ ___________________________________________________ </body> </html>
2B
Inspecciona el tráfico con DevTools
POST

Abre login.html en el navegador. Llena los campos con datos de prueba. Antes de enviar, abre las DevTools (F12 → pestaña Network / Red). Envía el formulario y observa la petición.

  • ¿Cambió la URL después de enviar el formulario? (Sí / No): _______
  • ¿Puedes ver la contraseña en la URL?: _______
  • En DevTools → Network, haz clic en la petición. ¿En qué sección encuentras los datos enviados?
Sección en DevTools: _____________________________________________________
💡 Pista

En la pestaña Network, selecciona la petición y busca la sección "Payload" o "Form Data". Ahí están los datos del POST.
