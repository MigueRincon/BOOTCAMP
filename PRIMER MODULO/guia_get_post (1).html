Métodos GET & POST
en Formularios HTML

Así es como los datos viajan desde el navegador hasta el servidor — la base de toda la web.

01 ¿Qué hace un formulario HTML?

Un formulario HTML (<form>) es el mecanismo principal para que el usuario envíe datos al servidor. Cuando el usuario hace clic en "enviar", el navegador toma todos los campos y los empaqueta según el método indicado.

Dos atributos del <form> son clave:

atributo

action

¿A dónde se envían los datos? Es la URL del servidor que los recibirá.

atributo

method

¿Cómo se envían? Las opciones son GET o POST.

02 Método GET

Con GET, los datos del formulario se adjuntan directamente a la URL como parámetros de consulta (query string). Puedes verlos en la barra de direcciones del navegador.

🔍 ¿Qué es una query string?

Es la parte de la URL que empieza con ?. Contiene pares clave=valor separados por &.

Si un formulario GET tiene los campos nombre = Ana y ciudad = Bogotá, la URL resultante sería:

https://mi-servidor.com/buscar?nombre=Ana&ciudad=Bogot%C3%A1

Los datos son visibles en la URL. El servidor los recibe a través de esa misma URL.

formulario_get.html 
<!-- Formulario con método GET -->
<form action="buscar.php" method="GET">

  <label for="nombre">Nombre:</label>
  <input type="text" id="nombre" name="nombre">

  <label for="ciudad">Ciudad:</label>
  <input type="text" id="ciudad" name="ciudad">

  <button type="submit">Buscar</button>

</form>

<!-- Resultado en la URL del navegador:
     buscar.php?nombre=Ana&ciudad=Bogot%C3%A1  -->
💡 El atributo name es clave

El atributo name de cada <input> se convierte en el nombre del parámetro en la URL. Sin él, el campo no se envía.

03 Método POST

Con POST, los datos viajan en el cuerpo (body) de la petición HTTP, no en la URL. El usuario no puede verlos en la barra de direcciones.

🔒 POST no significa seguro por sí solo

Los datos siguen viajando en texto plano si no hay HTTPS. Para seguridad real se necesita cifrado (SSL/TLS). POST sólo los oculta de la vista del usuario.

La URL del servidor no cambia. Los datos van "por dentro" de la petición:

── URL (no cambia) ─────────────────────────────────
https://mi-servidor.com/login

── Body (invisible en URL, viaja por HTTP) ──────────
usuario=ana@correo.com&password=••••••••
formulario_post.html 
<!-- Formulario con método POST -->
<form action="login.php" method="POST">

  <label for="usuario">Correo:</label>
  <input type="email" id="usuario" name="usuario">

  <label for="password">Contraseña:</label>
  <input type="password" id="password" name="password">

  <button type="submit">Iniciar sesión</button>

</form>

<!-- La URL NO cambia. Los datos van en el body:
     usuario=ana@correo.com&password=secreto123  -->

05 Comparación rápida

Característica GET POST
¿Dónde van los datos? En la URL (query string) En el body de la petición HTTP
¿Son visibles? ✅ Sí, en la barra de dirección ❌ No se ven en la URL
¿Se puede guardar la URL? ✅ Sí (favoritos, compartir) ❌ No (la URL no cambia)
Límite de datos ~2000 caracteres (depende del browser) Sin límite práctico
Uso típico Búsquedas, filtros, consultas Login, registro, formularios de contacto
¿Contraseñas? ⚠️ Nunca — quedan en el historial ✅ Sí (con HTTPS)
📌 Regla de oro para elegir el método

¿Estás consultando / buscando información? → Usa GET. ¿Estás enviando / modificando datos (registro, login, formulario)? → Usa POST.
