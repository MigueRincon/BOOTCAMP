01 · Fundamento

¿Qué es HTML?

HTML (HyperText Markup Language) es el lenguaje de marcado con el que se construye la estructura de toda página web. No es un lenguaje de programación — no "calcula" ni "toma decisiones". Es más bien el esqueleto o el andamio de lo que ves en el navegador.

🏠
Analogía · La casa

HTML es como los planos y la estructura de una casa: columnas, paredes, puertas, ventanas. CSS es la decoración y pintura. JavaScript es la electricidad y la domótica que hace que las cosas se muevan. Sin la estructura (HTML), no hay casa.

📰
Analogía · El periódico

Un periódico tiene titular, subtítulo, cuerpo del artículo, imágenes y pie de foto. HTML hace exactamente eso: etiqueta qué parte del contenido es qué cosa, para que el navegador lo muestre correctamente.

💡

Dato clave: HTML fue creado en 1991 por Tim Berners-Lee en el CERN para compartir documentos entre científicos. La versión actual, HTML5, es el estándar desde 2014 y sigue evolucionando.

¿Cómo lo ve el navegador?

Cuando abres una página web, el navegador descarga un archivo .html, lo lee de arriba hacia abajo, y construye el DOM (árbol de elementos) que finalmente pinta en pantalla.

Flujo básico 
Tú escribes HTML  →  Navegador lo lee  →  Construye el DOM  →  Pinta la página

03 · Fundamentos

La estructura base de todo HTML

Todo archivo HTML válido tiene siempre la misma base. Es como el certificado de nacimiento de tu página.

index.html · Estructura mínima 
<!-- 1. DOCTYPE: dice qué versión de HTML usamos -->
<!DOCTYPE html>

<!-- 2. Raíz del documento -->
<html lang="es">

  <!-- 3. HEAD: datos de la página (no visibles) -->
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Mi primera página</title>
  </head>

  <!-- 4. BODY: todo lo que el usuario VE -->
  <body>
    <h1>Hola Mundo</h1>
    <p>Mi primera página web.</p>
  </body>

</html>
✉️
Analogía · La carta

El <head> es el sobre de la carta: tiene el remitente, destinatario y datos de envío, pero tú no lo lees. El <body> es el contenido de la carta: lo que realmente lees.

¿Qué hace cada parte?

1

<!DOCTYPE html>

Le dice al navegador "este archivo usa HTML5". Siempre va primero. Sin esto, el navegador entra en "modo raro" y puede mostrar mal tu página.

2

<html lang="es">

Envuelve todo el documento. El atributo lang ayuda a los lectores de pantalla y al SEO (posicionamiento en Google).

3

<meta charset="UTF-8">

Permite mostrar tildes (á, é, ñ) y caracteres especiales. ¡Sin esto, los acentos se ven como símbolos raros!

4

<meta name="viewport">

Hace que la página se vea bien en celulares. Sin esto, el móvil muestra la versión de escritorio reducida.

04 · Core

Etiquetas esenciales

Hay más de 100 etiquetas HTML, pero con estas 20 puedes construir el 90% de cualquier página. Haz clic en cada tarjeta para ver un ejemplo.

📝 Texto y jerarquía

<h1> — <h6>

Títulos y subtítulos. h1 es el más importante, h6 el menor.

<h1>Título principal</h1>
<h2>Subtítulo</h2>
<h3>Sub-subtítulo</h3>
<p>

Párrafo. Bloque de texto. El más usado de todos.

<p>Esto es un párrafo de texto normal.</p>
<strong>

Texto en negrita con significado semántico importante.

<p>Esto es <strong>muy importante</strong>.</p>
<em>

Énfasis. Texto en cursiva con significado.

<p>Esto es <em>énfasis</em>.</p>
<br>

Salto de línea. Es la única etiqueta sin cierre aquí.

<p>Línea 1<br>Línea 2</p>
<hr>

Línea horizontal divisoria entre secciones.

<p>Sección A</p>
<hr>
<p>Sección B</p>

📦 Contenedores

<div>

Contenedor en bloque. El más versátil. Para agrupar.

<div class="tarjeta">
<h2>Título</h2>
<p>Contenido</p>
</div>
<span>

Contenedor en línea. Para estilizar partes de texto.

<p>El precio es <span class="precio">$99</span></p>
⚠️

<div> vs <span>: <div> ocupa toda la línea (bloque). <span> solo ocupa lo que necesita (en línea). Como caja vs. palabra subrayada.

🧠 Quiz rápido
¿Qué etiqueta usarías para el título principal de una página?
<title>
<h1>
<p>
<strong>
✅ ¡Correcto! <h1> es el encabezado principal. Solo debe haber uno por página.
❌ Casi. <title> va en el <head> y aparece en la pestaña del navegador, no en la página. Para el contenido visible usamos <h1>.

05 · Atributos

Atributos: los datos extra

Los atributos dan información adicional a las etiquetas. Van dentro de la etiqueta de apertura con el formato nombre="valor".

👤
Analogía · La persona

Si una etiqueta es una persona, sus atributos son sus características: nombre, edad, color de ojos. La persona es <img>, su nombre (atributo) es src, y su valor es la foto en sí.

Atributo
¿Para qué sirve?
Ejemplo
id
Identificador único del elemento en la página
id="menu-principal"
class
Clase(s) para aplicar CSS. Puede repetirse
class="tarjeta activa"
href
URL de destino en enlaces <a>
href="https://google.com"
src
Ruta de la imagen en <img>
src="foto.jpg"
alt
Texto alternativo de imagen (accesibilidad)
alt="Foto de perfil"
target
Dónde abrir el enlace
target="_blank"
style
CSS en línea (evitar en proyectos reales)
style="color:red"
disabled
Desactiva un botón o input
disabled
placeholder
Texto de ejemplo en campos de formulario
placeholder="Tu email..."
⚠️

id vs class: id es como el número de cédula — único, irrepetible. class es como el uniforme del colegio — pueden usarlo muchos elementos a la vez.

06 · Contenido

Listas

Las listas son fundamentales para organizar información. Hay 3 tipos:

Los 3 tipos de listas 
<!-- Lista sin orden (bullets) -->
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

<!-- Lista ordenada (numerada) -->
<ol>
  <li>Abrir VS Code</li>
  <li>Crear index.html</li>
  <li>Escribir código</li>
</ol>

<!-- Lista de definición -->
<dl>
  <dt>HTML</dt>
  <dd>Lenguaje de marcado para estructurar páginas</dd>
</dl>

07 · Multimedia

Enlaces e imágenes

🔗 El enlace: <a>

La A viene de Anchor (ancla). Es lo que conecta páginas entre sí. Sin <a>, no existiría la web.

Tipos de enlaces 
<!-- Enlace externo (abre en nueva pestaña) -->
<a href="https://google.com" target="_blank">
  Ir a Google
</a>

<!-- Enlace interno (otra página del sitio) -->
<a href="contacto.html">Contacto</a>

<!-- Enlace a sección de la misma página -->
<a href="#sobre-mi">Sobre mí</a>

<!-- Enlace de email -->
<a href="mailto:hola@email.com">Escríbeme</a>

🖼️ La imagen: <img>

Las imágenes son etiquetas auto-cerradas (no necesitan </img>). El atributo alt es obligatorio por accesibilidad.

Imágenes 
<!-- Imagen local -->
<img src="fotos/mi-foto.jpg"
     alt="Mi foto de perfil"
     width="300"
     height="200">

<!-- Imagen desde URL -->
<img src="https://placekitten.com/300/200"
     alt="Un gatito adorable">

<!-- Imagen dentro de enlace -->
<a href="portfolio.html">
  <img src="proyecto1.jpg" alt="Proyecto 1">
</a>
💡

Nunca olvides el alt: El atributo alt es leído por los lectores de pantalla para personas con discapacidad visual. Además, Google lo usa para entender tus imágenes y posicionarlas en búsquedas.

08 · Datos

Tablas

Las tablas se usan para mostrar datos tabulares: horarios, comparativas, estadísticas. No para diseñar layouts (eso es CSS).

📊
Analogía · La hoja de Excel

Una tabla HTML es como una hoja de cálculo: tiene filas (<tr>), columnas de cabecera (<th>) y celdas de datos (<td>).

Tabla completa con thead / tbody / tfoot 
<table>

  <!-- Cabecera -->
  <thead>
    <tr>
      <th>Tecnología</th>
      <th>Creada en</th>
      <th>Uso principal</th>
    </tr>
  </thead>

  <!-- Cuerpo de datos -->
  <tbody>
    <tr>
      <td>HTML</td>
      <td>1991</td>
      <td>Estructura de páginas web</td>
    </tr>
    <tr>
      <td>CSS</td>
      <td>1996</td>
      <td>Estilos y diseño visual</td>
    </tr>
  </tbody>

</table>

09 · Interacción

Formularios

Los formularios son la forma en que los usuarios envían información. Login, registro, búsqueda, contacto — todo es un formulario.

📋
Analogía · El formulario del banco

Un formulario HTML es como llenar un formulario físico: tiene campos para escribir, opciones para marcar y un botón para entregar. La diferencia es que los datos viajan a un servidor.

Formulario de registro completo 
<form action="/registro" method="POST">

  <!-- Campo de texto -->
  <label for="nombre">Nombre:</label>
  <input type="text"
         id="nombre"
         name="nombre"
         placeholder="Tu nombre completo"
         required>

  <!-- Email -->
  <label for="email">Email:</label>
  <input type="email" id="email" required>

  <!-- Contraseña -->
  <input type="password" id="pass" minlength="8">

  <!-- Selector -->
  <select name="rol">
    <option value="estudiante">Estudiante</option>
    <option value="profesional">Profesional</option>
  </select>

  <!-- Checkbox -->
  <input type="checkbox" id="terminos" required>
  <label for="terminos">Acepto los términos</label>

  <!-- Botón enviar -->
  <button type="submit">Registrarme</button>

</form>

10 · Semántica

HTML Semántico: código con significado

HTML5 trajo etiquetas con significado propio. En vez de poner todo en <div>, usamos etiquetas que describen qué es cada parte.

🏛️
Analogía · El edificio

Antes: todo era "cuarto". Ahora: hay recepción, sala, cocina, baño, dormitorio. Todos son cuartos, pero su nombre describe para qué sirven. HTML semántico hace lo mismo.

<header>
Logo · Nombre del sitio · Tagline
<nav>
Inicio · Sobre mí · Proyectos · Contacto
<main>
<section>
<article>
Entrada del blog · tarjeta de proyecto
<aside>
Sidebar · Contenido relacionado · Publicidad
<footer>
© 2025 · Links · Redes sociales
Etiqueta¿Para qué sirve?Equivalente antiguo
<header>Cabecera del sitio o sección<div id="header">
<nav>Navegación principal<div id="nav">
<main>Contenido principal (solo uno)<div id="main">
<section>Sección temática con título<div class="section">
<article>Contenido independiente (post, noticia)<div class="post">
<aside>Contenido relacionado (sidebar)<div id="sidebar">
<footer>Pie de página<div id="footer">
<figure>Imagen con caption<div class="img-wrap">
💡

¿Por qué importa? El HTML semántico mejora el SEO (Google lo entiende mejor), la accesibilidad (los lectores de pantalla navegan por secciones) y la mantenibilidad del código (tú y tus compañeros lo entenderán en 6 meses).

🧠 Quiz rápido
Tienes el menú de navegación de tu sitio. ¿Qué etiqueta semántica deberías usar?
<div id="menu">
<header>
<nav>
<section>
✅ ¡Exacto! <nav> es específicamente para bloques de navegación. Los motores de búsqueda y lectores de pantalla lo reconocen como el menú del sitio.
❌ Técnicamente funciona, pero lo correcto semánticamente es <nav>, que describe exactamente su propósito: navegación.

Casos de uso reales

HTML no es "un ejercicio académico". Está en absolutamente todo lo que usas en internet:

🛒
E-commerce
Tiendas online

Cada producto es un <article>, cada precio un <span>, el carrito un <aside>. Mercado Libre, Amazon — puro HTML estructurado.

📰
Noticias y blogs
Medios digitales

El New York Times usa <article> para cada noticia, <header> con el título y <time> para la fecha exacta.

🎓
Plataformas educativas
EdTech

Coursera, Platzi, Duolingo: formularios de progreso, videos <video>, listas de lecciones <ol> y cuestionarios con <form>.

💼
Portafolio personal
Tu carrera

Lo primero que revisará un reclutador es tu sitio web. Un portafolio en HTML + CSS te consigue el primer empleo.

📱
Apps web
PWA · SPA

Gmail, Google Docs, Figma en el navegador — todo es HTML. JavaScript añade interactividad, pero la estructura base es HTML.

📧
Emails HTML
Marketing

Los newsletters de tu marca favorita están escritos en HTML (con tablas, imágenes y enlaces). Mailchimp los renderiza.

12 · Profesionalismo

Buenas prácticas

Diferencia entre código que funciona y código profesional:

  • Siempre usar DOCTYPE y charset UTF-8
  • Un solo <h1> por página (el título principal)
  • Siempre incluir alt en las imágenes
  • Usar etiquetas semánticas en vez de <div> para todo
  • Indentar correctamente (Tab = 2 espacios)
  • Los ids son únicos, las clases pueden repetirse
  • Atributo lang="es" en el <html>
  • No usar style="" en línea — usar clases CSS
  • Siempre cerrar las etiquetas que lo requieren
  • Validar el código en validator.w3.org
  • Usar minúsculas para todos los nombres de etiquetas y atributos
  • Comentar secciones complejas con <!-- -->
⚠️

Error más común: Usar <br><br> para crear espacio. Eso no es HTML, es CSS. Los márgenes y espacios se controlan siempre con margin y padding.
