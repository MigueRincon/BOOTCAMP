Bloque 1: La propiedad display

La propiedad display es una de las más importantes de CSS. Define cómo se comporta un elemento en el flujo del documento: ¿ocupa toda la línea? ¿se pone al lado de otros? ¿es invisible?

Valor Comportamiento ¿Acepta width/height? Ejemplos de etiquetas
block Ocupa toda la línea disponible Sí <div>, <p>, <h1>-<h6>, <section>
inline Solo ocupa lo que necesita su contenido No <span>, <a>, <strong>, <em>
inline-block En línea, pero acepta dimensiones Sí <img>, <button>, <input>
none El elemento desaparece completamente — Cualquiera (se oculta)

1.1 — display: block

Los elementos de bloque siempre empiezan en una nueva línea y se expanden para ocupar todo el ancho disponible de su contenedor padre. Piensa en ellos como "ladrillos" que se apilan uno debajo del otro.

Soy un <div> — Bloque 1
Soy un <div> — Bloque 2
Soy un <div> — Bloque 3
/* Así se comportan los elementos block por defecto */
div {
    display: block;     /* Ya es block por defecto */
    background: #FF6584;
    padding: 10px;
    margin: 6px 0;
}
💡 Dato clave: Aunque un <div> contenga solo una palabra, ocupará toda la línea. Para cambiar esto, necesitas cambiar su display.

1.2 — display: inline

Los elementos inline fluyen dentro del texto, uno al lado del otro. No aceptan width ni height, y los márgenes verticales no los afectan.

Aquí hay texto normal y elemento 1 elemento 2 elemento 3 y el texto sigue fluyendo naturalmente. 

span {
    display: inline;     /* Ya es inline por defecto */
    background: #6C63FF;
    padding: 4px 12px;

    /* ⚠️ ESTO NO FUNCIONARÁ en elementos inline: */
    width: 200px;       /* Ignorado */
    height: 100px;      /* Ignorado */
    margin-top: 50px;   /* Ignorado */
}
⚠️ Error común de principiante: Intentar ponerle width o height a un <span> o un <a> y no ver ningún cambio. ¡Recuerda que son inline!

1.3 — display: inline-block

Lo mejor de dos mundos: se comporta en línea (al lado de otros), pero acepta width y height como un bloque. Es perfecto para crear botones, tarjetas en fila o menús horizontales.

Caja A
Caja B
Caja C
.caja {
    display: inline-block;
    width: 140px;
    padding: 14px 24px;
    background: #43E97B;
    text-align: center;
}

1.4 — display: none

El elemento desaparece completamente: no se ve y no ocupa espacio. Es diferente a visibility: hidden, que oculta el elemento pero conserva su espacio.

Elemento 1 — Visible
Elemento 2 — display: none (no me ves)
Elemento 3 — Visible (el 2 no ocupa espacio)

Comparación con visibility: hidden:

Elemento 1 — Visible
Elemento 2 — visibility: hidden
Elemento 3 — Nota el espacio vacío arriba
/* Desaparece completamente */
.oculto {
    display: none;
}

/* Invisible pero ocupa espacio */
.invisible {
    visibility: hidden;
}

Bloque 2: La propiedad position

Por defecto, los elementos siguen el "flujo normal" del documento: de arriba abajo y de izquierda a derecha. La propiedad position nos permite sacar elementos de ese flujo y colocarlos donde queramos, usando las propiedades top, right, bottom y left.

Valor ¿Sale del flujo? ¿Se posiciona respecto a...? Uso típico
static No Nada (es el valor por defecto) Comportamiento normal
relative No Su posición original Ajustes pequeños, contenedor para absolute
absolute Sí Primer ancestro con position != static Badges, tooltips, overlays
fixed Sí La ventana del navegador (viewport) Headers fijos, botones flotantes
sticky No (hasta que se "pega") Su contenedor, al hacer scroll Menús que se pegan al hacer scroll

2.1 — position: static

Es el valor por defecto. El elemento sigue el flujo normal. Las propiedades top, left, etc. no tienen efecto.

2.2 — position: relative

El elemento mantiene su espacio en el flujo, pero puede moverse respecto a su posición original. Es como decir "muévete 20px hacia abajo desde donde estabas".

Normal
Relative (top: 20px, left: 40px)
Normal (nota el espacio vacío arriba)
.movido {
    position: relative;
    top: 20px;    /* Se mueve 20px hacia abajo */
    left: 40px;   /* Se mueve 40px hacia la derecha */
}

2.3 — position: absolute

El elemento sale del flujo y se posiciona respecto al ancestro más cercano que tenga position: relative (u otro valor distinto de static). Si no encuentra ninguno, se posiciona respecto al <body>.

Contenedor padre (position: relative)

NUEVO
Abajo-izquierda
Abajo-derecha
/* El padre DEBE tener position: relative */
.padre {
    position: relative;
}

/* El hijo se posiciona dentro del padre */
.badge {
    position: absolute;
    top: 10px;
    right: 10px;
}
💡 Regla de oro: Si usas position: absolute en un hijo, el padre necesita position: relative. Si lo olvidas, el elemento se posicionará respecto a toda la página.

2.4 — position: fixed

El elemento se posiciona respecto a la ventana del navegador y no se mueve aunque hagas scroll. Se usa para headers fijos, botones flotantes y barras de navegación. 

/* Un botón que siempre está abajo a la derecha */
.btn-flotante {
    position: fixed;
    bottom: 20px;
    right: 20px;
    background: #6C63FF;
    color: white;
    padding: 16px 24px;
    border-radius: 50px;
    z-index: 999;
}

2.5 — position: sticky

Es un híbrido: se comporta como relative hasta que el usuario hace scroll y llega a un punto definido (por ejemplo top: 0), y entonces se "pega" como fixed. Perfecto para encabezados de tabla o menús secundarios. 

.menu-sticky {
    position: sticky;
    top: 0;     /* Se pega cuando llega al borde superior */
    background: white;
    z-index: 10;
}
💡 Importante: Para que sticky funcione, el contenedor padre debe tener suficiente altura para permitir el scroll, y debes especificar al menos una dirección (top, bottom, etc.).

Bloque 3: Flexbox

Flexbox es un modelo de layout unidimensional que facilita alinear, distribuir y ordenar elementos en una fila o columna. Olvídate de los hacks con float y margin: auto: Flexbox resuelve todo eso de forma elegante.

💡 Concepto clave: En Flexbox hay dos roles: el contenedor (padre, con display: flex) y los items (hijos directos). Las propiedades se dividen entre ambos. 
/* Activar Flexbox: solo necesitas esto en el padre */
.contenedor {
    display: flex;
}

3.1 — flex-direction

Define la dirección del eje principal: ¿los items van en fila o en columna?

flex-direction: row (por defecto)

💡 El truco más usado en CSS: Para centrar algo perfecta y completamente (horizontal y vertical): 
.centrado-perfecto {
    display: flex;
    justify-content: center;   /* Centra horizontalmente */
    align-items: center;       /* Centra verticalmente */
    height: 100vh;             /* Necesitas una altura definida */
}

Proyecto Integrador: Landing Page de una App

Desafiante

Pon en práctica todo lo que aprendiste creando una landing page completa. Este proyecto combina display, position y flexbox en un escenario real.

Diseño
Requisitos:
  1. Navbar (sticky) — Logo a la izquierda, enlaces a la derecha. Usa display: flex, justify-content: space-between, align-items: center. Hazlo position: sticky con top: 0.
  2. Hero section — Dos columnas con Flexbox: texto a la izquierda (título, subtítulo, botón CTA) e imagen/mockup a la derecha. Centra verticalmente con align-items: center.
  3. Sección Features — 3 tarjetas en fila con Flexbox. Cada tarjeta tiene un icono (usa emojis), título y descripción. Las tarjetas deben hacer wrap en pantallas pequeñas.
  4. Sección Precios — 3 tarjetas de precio en fila. La del medio es "destacada" y tiene un badge "Popular" con position: absolute.
  5. Footer — 3 columnas con Flexbox (reutiliza lo del ejercicio 5).
  6. Botón flotante — Un botón "Volver arriba" con position: fixed en la esquina inferior derecha.
💡 Consejos: No intentes hacerlo todo perfecto de una vez. Primero crea la estructura HTML completa, luego ve agregando CSS sección por sección. Empieza por el navbar, luego el hero, y así sucesivamente.

Cheat Sheet: Resumen rápido

Display

PropiedadEfecto
display: blockOcupa toda la línea, acepta width/height
display: inlineFluye con el texto, NO acepta width/height
display: inline-blockEn línea pero acepta width/height
display: noneDesaparece completamente
display: flexActiva Flexbox en el contenedor

Position

PropiedadEfecto
position: staticValor por defecto, flujo normal
position: relativeSe mueve respecto a sí mismo, mantiene espacio
position: absoluteSale del flujo, se posiciona respecto al padre relative
position: fixedFijo en la ventana, no se mueve con scroll
position: stickySe pega al hacer scroll

Flexbox

PropiedadEfecto
flex-directionrow | column | row-reverse | column-reverse
justify-contentAlinea en el eje principal
align-itemsAlinea en el eje cruzado
flex-wrapPermite saltar de línea
gapEspacio entre items
flex: 1El item crece para llenar espacio disponible
