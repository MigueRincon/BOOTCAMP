1

Que es CSS y por que importa

⏰ ~12 minutos
🎨 Analogia

Imagina que HTML es el esqueleto de una persona: la estructura basica, los huesos. CSS es la ropa, el peinado, el maquillaje. Sin CSS, todos los sitios web se verian como documentos de Word de los 90s.

📊 Dato curioso

CSS fue propuesto en 1996 por Hakon Wium Lie. Antes de eso, los desarrolladores usaban tablas de HTML para hacer layout... y era una pesadilla. Hoy, mas del 97% de los sitios web usan CSS.

CSS significa Cascading Style Sheets (Hojas de Estilo en Cascada). La palabra "cascada" es clave y la entenderemos mas adelante. Por ahora, piensa en CSS como el lenguaje que le dice al navegador como se ve cada elemento.

📚 Principio #1: Separacion de responsabilidades

HTML se encarga de la estructura, CSS del estilo visual, y JavaScript del comportamiento. Mezclarlos es como poner la cocina en el bano: funciona, pero nadie quiere vivir asi.

Hay tres formas de agregar CSS a una pagina:

1. Inline - Directamente en el atributo style del elemento. Rapido pero dificil de mantener.

2. Interno (<style>) - Dentro de una etiqueta <style> en el <head>. Util para paginas unicas.

3. Externo (archivo .css) - Un archivo separado enlazado con <link>. La forma profesional y recomendada.

Ejemplo de CSS inline (NO recomendado para proyectos reales):

<p style="color: red; font-size: 20px;">Hola!</p>

Es como escribir notas en los margenes de un libro: funciona para cosas rapidas, pero se vuelve caos.

Archivo externo estilos.css:

p { color: blue; font-size: 18px; }

Enlazado desde HTML:

<link rel="stylesheet" href="estilos.css">

Selectores: Apuntando con precision

⏰ ~15 minutos
🎨 Analogia

Los selectores CSS son como direcciones postales. Si quieres enviar un paquete (estilo) a una casa (elemento), necesitas una direccion precisa. Cuanto mas especifica, menos probabilidad de que llegue al vecino equivocado.

📚 Principio #2: Especificidad

Cuando varios selectores apuntan al mismo elemento, gana el mas especifico. El orden de poder es: inline > #id > .clase > elemento. Es como en un juego de cartas: el ID es el as, la clase es una reina, y el elemento es un numero bajo.

p { color: blue; }

Selecciona TODOS los parrafos. Es como decir "todos los gatos del mundo".

.destacado { background: yellow; }

Selecciona elementos con class="destacado". Es como decir "los gatos con collar rojo".

#titulo-principal { font-size: 3rem; }

Selecciona UN elemento unico con id="titulo-principal". Es como decir "el gato llamado Michi".

nav .menu li a:hover { color: red; }

Combinaciones avanzadas. Esto dice: "dentro de un nav, busca un elemento con clase menu, y dentro de un li, cambia el color del link cuando le pases el mouse".

Box Model: Todo es una caja

⏰ ~15 minutos
🎨 Analogia

Piensa en un regalo de cumpleanos: el contenido es el regalo, el padding es el relleno protector, el border es la caja de carton, y el margin es el espacio entre tu regalo y los demas en la mesa.

📚 Principio #3: Todo en CSS es una caja

Cada elemento HTML genera una caja rectangular. Entender el Box Model es como entender la gravedad en fisica: todo lo demas se construye sobre esto.

🎮 Laboratorio del Box Model

📊 Dato clave

Por defecto, width solo cuenta el contenido. Si le sumas padding y border, la caja crece! Para evitar eso, usa box-sizing: border-box y el width incluira todo. Casi todos los proyectos modernos usan * { box-sizing: border-box; }.

Colores y Tipografia

⏰ ~15 minutos
🎨 Analogia

Los colores en CSS son como la paleta de un pintor. Puedes mezclarlos de muchas formas: por nombre ("red"), por codigo hex (#FF0000), por RGB (rgb(255,0,0)), o por HSL (hsl(0, 100%, 50%)). HSL es el mas intuitivo: Hue (color), Saturation (intensidad), Lightness (brillo).

📚 Principio #4: Jerarquia visual con tipografia

La tipografia crea jerarquia. Usa font-size, font-weight, line-height y letter-spacing para guiar el ojo del usuario. Un buen truco: no uses mas de 2-3 fuentes diferentes en un proyecto.

Flexbox: Layout sin dolor

⏰ ~20 minutos
🎨 Analogia

Flexbox es como organizar libros en un estante. Puedes decidir: ponerlos todos a la izquierda, centrarlos, repartir el espacio entre ellos, o incluso apilarlos verticalmente. El estante es el contenedor flex y los libros son los items flex.

📚 Principio #5: Flexbox piensa en un eje

Flexbox trabaja en un eje principal (por defecto horizontal) y un eje secundario. Con justify-content controlas el eje principal, con align-items controlas el secundario. Es tu control remoto del layout.

📊 Dato pro

El truco mas comun de Flexbox: para centrar algo perfectamente, usa display: flex; justify-content: center; align-items: center; en el padre. Centrar en CSS solia ser una pesadilla, Flexbox lo resolvio en 3 lineas.

CSS Grid: Layout 2D como un pro

⏰ ~15 minutos
🎨 Analogia

Si Flexbox es organizar en una linea (como un estante), Grid es organizar en una tabla/cuadricula (como un tablero de ajedrez). Puedes definir filas Y columnas al mismo tiempo, y colocar elementos donde quieras.

📚 Principio #6: Flexbox vs Grid

Flexbox es ideal para layouts en 1 dimension (una fila O una columna). Grid es ideal para layouts en 2 dimensiones (filas Y columnas a la vez). No compiten, se complementan. Usa ambos segun lo necesites.

Transiciones, Animaciones y Responsive

⏰ ~15 minutos
🎨 Analogia

Una transicion es como encender un dimmer de luz: va suavemente de un estado a otro. Una animacion con @keyframes es como una coreografia completa: multiples pasos, repeticiones y timing personalizado.

🎮 Laboratorio de Transiciones

Pasa el mouse sobre el elemento para ver la transicion:

📚 Principio #7: Mobile First

Disenha primero para pantallas pequenas (movil) y luego agrega estilos para pantallas mas grandes con @media queries. Es como construir una casa: empieza con los cimientos solidos y luego agrega pisos.

🔥

Reto de Investigacion #7

Investiga:

  • Que son los breakpoints mas usados para responsive design? (pista: 768px, 1024px...)
  • Que diferencia hay entre min-width y max-width en media queries?
  • Que es clamp() y por que es el futuro del responsive?
8

Variables CSS y Proyecto Final

⏰ ~15 minutos
🎨 Analogia

Las variables CSS (custom properties) son como tener un panel de control central. En vez de cambiar "azul" en 50 lugares, lo defines una vez como --color-primario: blue y lo usas en todas partes con var(--color-primario). Cambias uno, cambian todos.

📚 Principio #8: DRY (Don't Repeat Yourself)

Las variables CSS te ayudan a seguir el principio DRY. Cuando te encuentres copiando y pegando el mismo color o valor, es hora de hacer una variable. Tu yo del futuro te lo agradecera.

🏆 Proyecto Final: Tu Tarjeta de Presentacion

Usa todo lo aprendido para crear una tarjeta de presentacion personal con HTML y CSS. Tu checklist:

  • Usar un archivo CSS externo
  • Incluir al menos 2 variables CSS
  • Usar Flexbox o Grid para el layout
  • Agregar una transicion o animacion
  • Usar Google Fonts para la tipografia
  • Que se vea bien en movil (responsive)
  • Usar box-sizing: border-box globalmente
  • Al menos 3 selectores diferentes (elemento, clase, combinado)
