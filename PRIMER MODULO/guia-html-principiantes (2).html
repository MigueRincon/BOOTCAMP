Nivel 1

Básico

01
Estructura base de un documento HTML
Aprende los huesos de cualquier página web: DOCTYPE, html, head y body.
Objetivo
Crea tu primer archivo HTML válido con todos los elementos obligatorios. Ábrelo en el navegador y verifica que se muestra correctamente.
Código de ejemplo
html 
<!-- Todo documento HTML empieza aquí -->
<!DOCTYPE html>
<html lang="es">

  <head>
    <meta charset="UTF-8"/>
    <meta name="viewport" content="width=device-width, initial-scale=1.0"/>
    <title>Mi primera página</title>
  </head>

  <body>
    ¡Hola, mundo!
  </body>

</html>
💡 Conceptos clave
  • <!DOCTYPE html> le dice al navegador qué versión de HTML usar (HTML5).
  • <head> contiene metadatos: no es visible para el usuario.
  • <body> contiene todo lo que el usuario ve.
  • El atributo lang mejora la accesibilidad y el SEO.
02
Títulos y párrafos
Usa las etiquetas h1–h6 y p para organizar texto con jerarquía visual.
Objetivo
Crea una página sobre tu tema favorito usando al menos 3 niveles de título y 2 párrafos con texto real.
Código de ejemplo
html 
<body>

  <h1>El Sistema Solar</h1>
  <p>El sistema solar se formó hace unos 4,600 millones de años.</p>

  <h2>Los planetas interiores</h2>
  <p>Mercurio, Venus, Tierra y Marte son los más cercanos al Sol.</p>

  <h3>La Tierra</h3>
  <p>Es el único planeta conocido con vida.</p>

</body>
💡 Reglas importantes
  • Solo debe haber un <h1> por página (título principal).
  • No saltes niveles: no pongas un <h3> después de un <h1> sin <h2>.
  • <p> crea espaciado automático entre párrafos.
🔥 Reto extra

Añade un párrafo con la etiqueta <strong> para texto en negrita y <em> para cursiva. Experimenta combinándolos.

03
Listas ordenadas y no ordenadas
Organiza información con <ul>, <ol> y elementos <li>.
Objetivo
Crea una receta de cocina: los ingredientes como lista no ordenada y los pasos como lista ordenada.
Código de ejemplo
html 
<h2>Ingredientes</h2>
<ul>
  <li>2 tazas de harina</li>
  <li>1 taza de leche</li>
  <li>3 huevos</li>
</ul>

<h2>Preparación</h2>
<ol>
  <li>Mezcla la harina y la leche.</li>
  <li>Agrega los huevos uno a uno.</li>
  <li>Hornea por 30 minutos a 180°C.</li>
</ol>

<!-- Lista anidada -->
<ul>
  <li>Frutas
    <ul>
      <li>Manzana</li>
      <li>Naranja</li>
    </ul>
  </li>
</ul>
💡 Diferencias clave
  • <ul> = Unordered List → muestra viñetas (bullets).
  • <ol> = Ordered List → muestra números automáticos.
  • Puedes anidar listas dentro de otras listas.
04
Enlaces e imágenes
Conecta páginas con <a> e inserta imágenes con <img>.
Objetivo
Crea una página con un enlace a Google que abra en una pestaña nueva y una imagen con su descripción alternativa.
Código de ejemplo
html 
<!-- ENLACES -->
<a href="https://www.google.com">Ir a Google</a>

<!-- Abre en pestaña nueva -->
<a href="https://www.wikipedia.org"
   target="_blank"
   rel="noopener noreferrer">
  Abrir Wikipedia
</a>

<!-- Enlace interno (a otra sección) -->
<a href="#contacto">Ir a Contacto</a>

<!-- IMÁGENES -->
<img
  src="https://picsum.photos/400/200"
  alt="Imagen aleatoria de ejemplo"
  width="400"
  height="200"
/>
💡 Atributos esenciales
  • href en <a> = la URL de destino. Puede ser absoluta o relativa.
  • target="_blank" abre en pestaña nueva. Siempre acompáñalo de rel="noopener".
  • alt en <img> es obligatorio: describe la imagen para lectores de pantalla y SEO.
  • Especificar width y height evita saltos de layout al cargar.
🔥 Reto extra

Convierte una imagen en un enlace: envuelve <img> dentro de <a>. Haz clic en la imagen para navegar a otro sitio.

Nivel 2

Medio

05
Tablas HTML
Muestra datos tabulares con <table>, thead, tbody y las celdas th / td.
Objetivo
Crea una tabla con el horario de tu semana: días en las columnas y materias/actividades en las filas.
Código de ejemplo
html 
<table border="1">

  <thead>
    <tr>
      <th>Día</th>
      <th>Actividad</th>
      <th>Hora</th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>Lunes</td>
      <td>Programación</td>
      <td>9:00 AM</td>
    </tr>
    <tr>
      <td>Martes</td>
      <td colspan="2">Libre (celda combinada)</td>
    </tr>
  </tbody>

  <tfoot>
    <tr>
      <td colspan="3">Total: 5 días</td>
    </tr>
  </tfoot>

</table>
💡 Anatomía de una tabla
  • <tr> = Table Row (fila).
  • <th> = Table Header (celda encabezado, en negrita).
  • <td> = Table Data (celda de datos).
  • colspan y rowspan combinan celdas horizontal y verticalmente.
06
Divs, Spans y clases
Agrupa y selecciona elementos con contenedores genéricos y atributos de clase.
Objetivo
Crea una tarjeta de perfil usando <div> como contenedor y <span> para resaltar partes del texto. Asigna clases descriptivas.
Código de ejemplo
html 
<!-- div es bloque, span es en línea -->
<div class="tarjeta">

  <div class="tarjeta-header">
    <img src="foto.jpg" alt="Avatar"/>
    <h2>Ana García</h2>
  </div>

  <div class="tarjeta-body">
    <p>
      Desarrolladora con 
      <span class="destacado">5 años de experiencia</span>
       en HTML y CSS.
    </p>
  </div>

</div>

<!-- Múltiples clases -->
<div class="caja grande activa">...</div>
💡 div vs span
  • <div> es un bloque: ocupa toda la línea.
  • <span> es en línea: solo ocupa su contenido.
  • Las clases son el puente entre HTML y CSS/JavaScript.
  • Un elemento puede tener múltiples clases separadas por espacios.
07
Formulario básico
Recopila datos del usuario con <form>, <input>, <label> y <button>.
Objetivo
Crea un formulario de registro con nombre, email, contraseña y un botón de envío. Usa labels correctamente.
Código de ejemplo
html 
<form action="/registro" method="post">

  <!-- label for="id" conecta con input id="id" -->
  <label for="nombre">Nombre completo</label>
  <input
    type="text"
    id="nombre"
    name="nombre"
    placeholder="Ej: María López"
    required
  />

  <label for="email">Correo electrónico</label>
  <input type="email" id="email" name="email" required/>

  <label for="pass">Contraseña</label>
  <input type="password" id="pass" name="pass"
         minlength="8" required/>

  <button type="submit">Registrarse</button>

</form>
💡 Tipos de input más usados
  • type="text" — texto libre.
  • type="email" — valida formato de email automáticamente.
  • type="number" — solo números.
  • type="checkbox" / type="radio" — opciones.
  • required hace el campo obligatorio sin JavaScript.
🔥 Reto extra

Agrega un campo select (desplegable) con opciones de país, y un textarea para una biografía corta.

08
Select, Checkbox y Radio
Implementa los controles de selección más comunes en formularios web.
Objetivo
Crea una encuesta de preferencias con un desplegable, checkboxes para intereses y radios para nivel de experiencia.
Código de ejemplo
html 
<!-- SELECT (una sola opción) -->
<label for="pais">País</label>
<select id="pais" name="pais">
  <option value="">— Selecciona —</option>
  <option value="mx">México</option>
  <option value="co">Colombia</option>
  <option value="ar">Argentina</option>
</select>

<!-- CHECKBOXES (múltiple selección) -->
<fieldset>
  <legend>¿Qué tecnologías conoces?</legend>
  <input type="checkbox" id="html" name="tech" value="html"/>
  <label for="html">HTML</label>
  <input type="checkbox" id="css" name="tech" value="css"/>
  <label for="css">CSS</label>
</fieldset>

<!-- RADIO (una sola opción del grupo) -->
<fieldset>
  <legend>Nivel</legend>
  <input type="radio" id="basico" name="nivel" value="basico"/>
  <label for="basico">Principiante</label>
  <input type="radio" id="avanzado" name="nivel" value="avanzado"/>
  <label for="avanzado">Avanzado</label>
</fieldset>
💡 Nota importante
  • Los radio del mismo grupo deben tener el mismo atributo name.
  • <fieldset> agrupa controles relacionados y <legend> los etiqueta.
  • El atributo value es lo que se envía al servidor, no el texto del label.

Intermedio

09
HTML Semántico
Reemplaza divs genéricos con elementos que describen su significado.
Objetivo
Reestructura una página de blog usando solo etiquetas semánticas: header, nav, main, article, aside y footer.
Código de ejemplo
html 
<header>
  <h1>Mi Blog de Tecnología</h1>
  <nav>
    <ul>
      <li><a href="/">Inicio</a></li>
      <li><a href="/blog">Blog</a></li>
    </ul>
  </nav>
</header>

<main>
  <article>
    <header>
      <h2>¿Qué es HTML semántico?</h2>
      <time datetime="2025-01-15">15 de enero 2025</time>
    </header>
    <section>
      <p>El HTML semántico usa etiquetas con significado...</p>
    </section>
    <footer>
      <p>Autor: Ana García</p>
    </footer>
  </article>

  <aside>
    <h3>Artículos relacionados</h3>
  </aside>
</main>

<footer>
  <p>&copy; 2025 Mi Blog</p>
</footer>
💡 ¿Por qué importa?
  • SEO: los buscadores entienden mejor tu contenido.
  • Accesibilidad: los lectores de pantalla navegan por landmarks semánticos.
  • <article> = contenido autónomo (podría existir solo).
  • <section> = sección temática dentro de otro elemento.
  • <aside> = contenido tangencial (barras laterales, notas).
10
Atributos de datos y accesibilidad
Usa data-*, aria-label y role para páginas más robustas e inclusivas.
Objetivo
Crea un botón de "Me gusta" con un atributo data-id, y un menú de navegación con atributos aria correctos.
Código de ejemplo
html 
<!-- data-* guarda datos personalizados para JS -->
<button
  data-id="post-42"
  data-accion="like"
  aria-label="Me gusta este artículo"
  aria-pressed="false"
>
  ❤️ Me gusta
</button>

<!-- Navegación accesible -->
<nav aria-label="Menú principal">
  <ul role="menubar">
    <li role="menuitem"><a href="/">Inicio</a></li>
  </ul>
</nav>

<!-- img decorativa (ocultar del lector) -->
<img src="decoracion.png" alt="" role="presentation"/>

<!-- Leer con JS: -->
<!-- const btn = document.querySelector('[data-id="post-42"]') -->
<!-- console.log(btn.dataset.id) // "post-42" -->
💡 data-* y ARIA
  • data-* almacena cualquier dato en el HTML, accesible desde JavaScript con element.dataset.
  • aria-label da nombre a elementos sin texto visible.
  • aria-pressed indica si un botón toggle está activo.
  • alt="" en imágenes decorativas las oculta de lectores de pantalla.
11
Audio, Video e iframes
Integra multimedia e iframes de manera accesible con atributos correctos.
Objetivo
Crea una página multimedia que incluya audio con controles, video con subtítulos y un mapa de Google embebido.
Código de ejemplo
html 
<!-- AUDIO -->
<audio controls preload="metadata">
  <source src="musica.mp3" type="audio/mpeg"/>
  <source src="musica.ogg" type="audio/ogg"/>
  Tu navegador no soporta audio.
</audio>

<!-- VIDEO con subtítulos -->
<video controls width="640" poster="miniatura.jpg">
  <source src="video.mp4" type="video/mp4"/>
  <track
    kind="subtitles"
    src="subtitulos-es.vtt"
    srclang="es"
    label="Español"
    default
  />
</video>

<!-- IFRAME (ej. YouTube) -->
<iframe
  src="https://www.youtube.com/embed/TU_VIDEO_ID"
  title="Título del video para accesibilidad"
  width="560"
  height="315"
  allowfullscreen
  loading="lazy"
></iframe>
💡 Buenas prácticas multimedia
  • Siempre ofrece múltiples formatos con <source> por compatibilidad.
  • poster muestra una imagen antes de reproducir el video.
  • <track> con subtítulos es esencial para accesibilidad.
  • loading="lazy" en iframes mejora la velocidad de carga.
12
Proyecto final: Página personal completa
Integra todo lo aprendido en una página de portafolio personal bien estructurada.
Objetivo
Construye una página de portafolio personal que incluya: navegación con anclas, sección "sobre mí", lista de habilidades, tabla de proyectos, galería de imágenes, formulario de contacto y footer con links a redes sociales.
Estructura recomendada
html — estructura 
<body>

  <header>
    <h1>Tu Nombre</h1>
    <nav>
      <a href="#sobre-mi">Sobre mí</a>
      <a href="#proyectos">Proyectos</a>
      <a href="#contacto">Contacto</a>
    </nav>
  </header>

  <main>
    <section id="sobre-mi">
      <img src="tu-foto.jpg" alt="Foto de perfil"/>
      <p>Descripción sobre ti...</p>
    </section>

    <section id="proyectos">
      <h2>Proyectos</h2>
      <table>...</table>
    </section>

    <section id="contacto">
      <h2>Contacto</h2>
      <form method="post">...</form>
    </section>
  </main>

  <footer>
    <p>&copy; 2025 Tu Nombre</p>
    <a href="https://github.com/tu-usuario"
       target="_blank"
       rel="noopener">GitHub</a>
  </footer>

</body>
✅ Checklist del proyecto
  • Estructura semántica completa (header, main, footer).
  • Al menos un formulario funcional con labels.
  • Una tabla con thead, tbody y tfoot.
  • Todos los links con href y todas las imágenes con alt.
  • El documento pasa el validador: validator.w3.org
  • La página es accesible navegando solo con el teclado (Tab).
