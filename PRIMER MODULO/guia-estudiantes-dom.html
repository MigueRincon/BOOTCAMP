La guía
Bonus
Introducción
¿Qué vas a aprender hoy?
Vamos a construir un contador de puntos que funciona en vivo en el navegador. Parece simple, pero en el camino vas a aprender los conceptos más importantes del desarrollo web.
qué vas a construir
🎮
Contador de puntos interactivo
HTML + CSS + JavaScript trabajando juntos
Dos botones que suman y restan puntos. Un número que cambia en tiempo real. Un mensaje que aparece cuando llegás a 10 puntos. Tarjetas de jugadores que se resaltan. Todo sin recargar la página.
los 3 roles — qué hace cada tecnología
  • H
    HTML — es la estructura. Define qué elementos existen en la página: el título, los botones, los párrafos. Es como el esqueleto de un edificio.
  • C
    CSS — es el diseño. Decide cómo se ve todo: colores, tamaños, fuentes, espaciado. Es la pintura y decoración del edificio.
  • J
    JavaScript — es el comportamiento. Hace que las cosas pasen: botones que responden, números que cambian, elementos que aparecen y desaparecen. Es la electricidad del edificio.
    • el DOM — el concepto más importante
    🗺️
    ¿Qué es el DOM?
    Document Object Model
    Cuando el navegador carga tu HTML, lo convierte en un mapa interno de todos los elementos. Ese mapa se llama DOM (Document Object Model). JavaScript puede entrar a ese mapa y cambiar cualquier cosa: texto, colores, visibilidad — todo en tiempo real, sin recargar.
    Analogía: Imaginá que el HTML es un árbol con ramas y hojas. El DOM es ese mismo árbol pero dibujado como un mapa que JavaScript puede leer y modificar. Si JS cambia una hoja en el mapa, la página se actualiza al instante.
    💡
    Para verlo en vivo: abrí cualquier página web, hacé clic derecho → "Inspeccionar" → pestaña "Elements". Ese árbol que ves es el DOM. ¡Podés hacer clic en cualquier elemento y editarlo en tiempo real!
    Antes de empezar
    Preparar tu espacio de trabajo
    Antes de escribir código, necesitás tener todo en orden. Siguiendo estos pasos no vas a tener problemas de "¿por qué no funciona?".
    paso a paso — hacelo ahora
    1. 1
      Instalá VS Code si no lo tenés. Buscalo en Google como "Visual Studio Code" o andá a code.visualstudio.com. Es gratis y es el editor más usado en el mundo.
    2. 2
      Instalá la extensión Live Server. Abrí VS Code → presioná Ctrl + Shift + X → buscá "Live Server" de Ritwick Dey → instalá. Esta extensión recarga el navegador automáticamente cada vez que guardás.
    3. 3
      Creá una carpeta en tu escritorio o documentos. Llamala mi-contador. Nada de espacios ni tildes en el nombre.
    4. 4
      Abrí esa carpeta con VS Code. Menú Archivo → "Abrir carpeta" → elegís mi-contador.
    5. 5
      Creá 3 archivos vacíos dentro de la carpeta. En el explorador de VS Code (panel izquierdo), hacé clic derecho → "Nuevo archivo" y creá:
      → index.html
      → style.css
      → script.js
    ⚠️
    Los 3 archivos tienen que estar en la misma carpeta. Si están en distintos lugares, no van a poder "verse" entre sí y nada va a funcionar.
    cómo usar live server
    ✅
    Para lanzar tu página: en el explorador de VS Code, hacé clic derecho sobre index.html → "Open with Live Server". Se va a abrir el navegador en localhost:5500. Cada vez que guardés con Ctrl+S, la página se recarga sola.
    Tarea tuya
    El HTML — vos lo escribís
    Abrí index.html y construí la estructura. Vos decidís cómo organizarlo, pero hay elementos específicos que necesitamos — con ids y clases exactos — para que el JavaScript los encuentre después.
    🔑
    Concepto clave — id vs class:
    — Un id es como el DNI: es único, solo un elemento puede tenerlo. Se usa cuando el elemento es especial.
    — Una class es como una etiqueta de grupo: muchos elementos pueden tenerla. Se usa cuando querés aplicar algo a varios a la vez.
    elementos que debes incluir
    1. A
      Un título cualquiera para tu página. No necesita id.
    2. B
      Un elemento (puede ser un <span> dentro de un <div>) con id="numero-puntos" que muestre el número 0.
    3. C
      Un párrafo <p> con id="mensaje" que diga algo como "¡Pulsá los botones!".
    4. D
      Un botón con el texto "Sumar" y el atributo onclick="sumarPunto()". Atención: los paréntesis son obligatorios.
    5. E
      Un botón con el texto "Restar" y el atributo onclick="restarPunto()".
    6. F
      Exactamente 3 divs, cada uno con class="jugador", con nombres de personas adentro.
    7. G
      Un div con id="ganador" y class="oculto" que diga "¡GANASTE!"
    referencia — así se ve una estructura posible
    HTML
    <!DOCTYPE html> <html lang="es"> <head> <meta charset="UTF-8"/> <title>Mi Contador</title> <link rel="stylesheet" href="style.css"/> </head> <body> <h1>Contador de Puntos</h1> <!-- El número: tiene id para que JS lo encuentre --> <div> <span id="numero-puntos">0</span> </div> <!-- Mensaje de estado --> <p id="mensaje">¡Pulsá los botones!</p> <!-- Botones — el onclick llama a funciones de JS --> <button onclick="sumarPunto()">➕ Sumar</button> <button onclick="restarPunto()">➖ Restar</button> <!-- Tarjetas de jugadores: misma clase los 3 --> <div class="jugador">Ana</div> <div class="jugador">Luis</div> <div class="jugador">María</div> <!-- Mensaje ganador: oculto por defecto --> <div id="ganador" class="oculto">🏆 ¡GANASTE!</div> <!-- El JS va AQUÍ, al final del body --> <script src="script.js"></script> </body> </html>
    ⚠️
    Error clásico: los ids son sensibles a mayúsculas y espacios. id="numero-puntos" es distinto a id="Numero Puntos" o id="numeroPuntos". Anotátelo bien porque JS va a buscar el id exacto.
    Tarea tuya
    El CSS — vos lo diseñás
    Abrí style.css. Acá tenés total libertad de diseño — colores, fuentes, espaciado, lo que quieras. Pero hay dos clases que son obligatorias porque JavaScript las va a activar y desactivar.
    las dos clases que JS necesita (obligatorias)
    CSS
    /* ─── CLASE 1: ocultar/mostrar elementos ───────── */ /* JS quita esta clase para mostrar el mensaje ganador */ .oculto { display: none; } /* ─── CLASE 2: resaltar tarjetas de jugadores ──── */ /* JS agrega esta clase cuando se gana */ .resaltado { border: 3px solid #22c55e; background-color: #f0fdf4; }
    💡
    El concepto importante: JavaScript no escribe CSS directamente. En cambio, agrega o quita clases. Vos definís el diseño en CSS; JS solo decide cuándo aplicarlo. Esta separación de responsabilidades es fundamental en el desarrollo web.
    ejemplo de diseño completo (opcional — podés hacerlo diferente)
    CSS
    /* Estilos generales */ body { font-family: sans-serif; background: #1a1a2e; color: #e0e0e0; text-align: center; padding: 40px 20px; } /* El número grande */ #numero-puntos { font-size: 5rem; font-weight: bold; color: #e94560; display: block; } /* Tarjetas de jugadores */ .jugador { background: #16213e; border: 1px solid #0f3460; border-radius: 10px; padding: 12px 20px; margin: 8px auto; max-width: 300px; transition: all 0.3s; } /* ── Las dos clases obligatorias ── */ .oculto { display: none; } .resaltado { border: 2px solid #2ecc71; background: #1a3a2e; }
    Guiado por el profe
    Conectar el JavaScript al HTML
    El HTML y el JS son archivos separados. Para que "se conozcan", el HTML necesita una instrucción que diga "cargá este archivo JS". Esa instrucción es la etiqueta <script>.
    dónde va la etiqueta script
    HTML — al final del body
    <body> <!-- ... todo tu HTML aquí arriba ... --> <!-- ESTO va AL FINAL, justo antes de </body> --> <script src="script.js"></script> </body>
    por qué va al final y no en el head
    Cuando el navegador carga una página, la lee de arriba hacia abajo. Si el <script> está en el <head>, JavaScript empieza a ejecutarse antes de que existan los elementos del HTML.

    Al ponerlo al final del body, todo el HTML ya existe cuando el script se ejecuta, y JS puede encontrar todos los elementos sin problemas.
    Analogía: Es como llegar a una fiesta e intentar presentarte con la gente. Si llegás antes que todos (script en el head), no hay nadie con quien hablar. Si llegás al final (script al final del body), todos ya están ahí y podés encontrarlos fácilmente.
    verificá que funciona
    ✅
    Abrí script.js y escribí esta única línea:

    console.log("¡JS conectado correctamente!")

    Guardá (Ctrl+S), abrí tu página con Live Server, presioná F12 → pestaña Consola. Si ves el mensaje, ¡todo está conectado!
    🔧
    La Consola (F12) es tu mejor amiga. Es donde aparecen los errores, los mensajes de console.log() y toda la información de depuración. Mantela siempre visible mientras programás.
    Guiado por el profe
    Las variables — la memoria de tu programa
    Antes de buscar nada en el DOM, necesitamos una variable que guarde el puntaje. Las variables son "cajitas" donde JavaScript guarda información que puede cambiar.
    cómo declarar variables
    script.js
    // "let" → el valor puede cambiar (los puntos suben y bajan) let puntos = 0; // "const" → el valor nunca cambia (la meta siempre es 10) const META = 10; // Comprobamos que JS funciona console.log("Puntos al inicio:", puntos); // → 0
    let vs const — cuándo usar cada uno
    Palabra clave ¿Puede cambiar? Cuándo usarla
    let Sí, puede cambiar Contadores, estados, valores que se actualizan
    const No, es fija Metas, configuraciones, valores que nunca cambian
    Analogía de la pizarra: la variable puntos es como tener el número en la cabeza (memoria interna). El DOM (lo que se ve en pantalla) es como la pizarra. Cuando cambiás la variable, tenés que actualizar la pizarra a mano — no se actualizan solas.
    ⚠️
    Error conceptual común: si solo cambiás el texto en el DOM pero no actualizás la variable, la lógica se pierde. Siempre: primero la variable, luego el DOM.
    Guiado por el profe
    Buscar elementos en el DOM
    Para que JS pueda cambiar un elemento HTML, primero tiene que encontrarlo. Hay dos métodos principales dependiendo de si buscás por id o por clase.
    método 1 — buscar por id
    script.js
    // document.getElementById("el-id") // → Devuelve UN elemento (porque los ids son únicos) const elNumero = document.getElementById("numero-puntos"); const elMensaje = document.getElementById("mensaje"); const elGanador = document.getElementById("ganador"); // Verificamos qué obtuvimos: console.log(elNumero); // → <span id="numero-puntos">0</span> // Si aparece null → el id en el HTML no coincide exactamente
    ⚠️
    Si la Consola muestra null: el id que escribiste en JS no existe exactamente igual en el HTML. Revisá mayúsculas, guiones y espacios. "numero-puntos" ≠ "NumeroPuntos".
    método 2 — buscar por clase (varios elementos)
    script.js
    // document.getElementsByClassName("la-clase") // → Devuelve una LISTA de elementos (HTMLCollection) // porque muchos elementos pueden tener la misma clase const jugadores = document.getElementsByClassName("jugador"); console.log(jugadores); // → HTMLCollection(3) [div, div, div] console.log(jugadores.length); // → 3 (cuántos encontró) console.log(jugadores[0]); // → primer div (índice empieza en 0) console.log(jugadores[1]); // → segundo div console.log(jugadores[2]); // → tercer div
    resumen de las diferencias
    Método Devuelve Cuándo usarlo
    getElementById Un solo elemento Cuando el elemento es único (contadores, mensajes, titulos)
    getElementsByClassName Una lista (HTMLCollection) Cuando querés agarrar un grupo del mismo tipo
    Analogía: getElementById es como buscar a alguien por su número de DNI — solo puede haber uno. getElementsByClassName es como llamar "¡estudiantes!" en el pasillo — todos los que tienen esa etiqueta levantan la mano.
    Guiado por el profe
    Las funciones — acciones que se ejecutan
    Una función es un bloque de código con nombre que se ejecuta cuando alguien lo llama. Los botones con onclick llaman funciones. Vamos a crear las dos funciones vacías primero, para verificar la conexión.
    estructura de una función
    script.js
    // Así se declara una función: function nombreDeLaFuncion() { // el código que se ejecuta va aquí adentro } // Así se llama (ejecuta) una función: nombreDeLaFuncion(); // ← los paréntesis son OBLIGATORIOS
    creá las funciones vacías primero
    script.js
    // Empezamos vacías para verificar que el botón conecta: function sumarPunto() { console.log("¡Botón SUMAR pulsado!"); } function restarPunto() { console.log("¡Botón RESTAR pulsado!"); }
    ✅
    Guardá (Ctrl+S) y hacé clic en el botón "Sumar". Abrí F12 → Consola. ¿Aparece el mensaje? Perfecto. Si no aparece, revisá que el nombre en onclick="sumarPunto()" sea idéntico al nombre de la función en el JS.
    💡
    Técnica profesional: siempre empezá con funciones vacías que solo hacen console.log(). Así confirmás que la conexión funciona antes de escribir lógica. Si el log no aparece, sabés exactamente dónde está el problema.
    Guiado por el profe
    Lógica con if / else
    Ahora llenás las funciones con código real. El if / else permite que el programa tome decisiones: "si pasa X, hacé A; si no, hacé B".
    cómo funciona if / else
    Estructura
    if (condición que puede ser verdadera o falsa) { // esto se ejecuta si la condición es VERDADERA } else { // esto se ejecuta si la condición es FALSA }
    función sumarPunto — completa
    script.js
    function sumarPunto() { // 1. Actualizamos la variable interna puntos = puntos + 1; // también se puede escribir: puntos++ // 2. Reflejamos el nuevo valor en el DOM elNumero.textContent = puntos; // .textContent cambia el texto del elemento // 3. Lógica: ¿llegamos a la meta? if (puntos >= META) { elMensaje.textContent = "🎉 ¡Ganaste!"; } else { elMensaje.textContent = "Vas por " + puntos + " puntos"; } }
    🔥
    Guardá y probá pulsar "Sumar" varias veces. El número en pantalla debe cambiar en tiempo real. ¡Eso es el DOM en acción!
    función restarPunto — con validación
    script.js
    function restarPunto() { // Validación: no dejamos bajar de 0 if (puntos > 0) { puntos = puntos - 1; // también se puede: puntos-- elNumero.textContent = puntos; elMensaje.textContent = "Bajaste a " + puntos + " puntos"; } else { elMensaje.textContent = "⚠️ ¡No podés bajar de 0!"; } }
    🤔
    ¿Por qué el if de validación? Sin él, los puntos podrían llegar a -1, -2, etc. La validación es un "guardián" que protege la lógica. En programación, anticipar casos problemáticos se llama manejo de casos borde.
    Guiado por el profe
    Manipular clases desde JavaScript
    Ahora conectás todo: JS va a agregar y quitar las clases CSS que creaste antes. Así mostrás el banner ganador y resaltás los jugadores.
    los 3 métodos de classList
    .classList.add("clase")
    Agrega una clase al elemento. Si ya la tiene, no hace nada.
    .classList.remove("clase")
    Quita una clase del elemento. Si no la tiene, no hace nada.
    .classList.toggle("clase")
    Si la tiene la quita, si no la tiene la agrega. Como un interruptor.
    .classList.contains("clase")
    Devuelve true o false según si el elemento tiene esa clase.
    mostrar el banner ganador
    script.js — dentro del if (puntos >= META)
    if (puntos >= META) { elMensaje.textContent = "🎉 ¡Ganaste!"; // Quitamos la clase "oculto" → el elemento aparece elGanador.classList.remove("oculto"); }
    resaltar los jugadores con un bucle for
    script.js — también dentro del if
    // "jugadores" es una lista de 3 elementos // Para hacer algo con CADA UNO, usamos un bucle for: for (let i = 0; i < jugadores.length; i++) { jugadores[i].classList.add("resaltado"); } // El for funciona así: // i empieza en 0 // mientras i sea menor que la cantidad de jugadores... // ejecuta el código del bloque y luego suma 1 a i
    Analogía del bucle: es como si tuvieras una lista de tarea en papel y fueras marcando cada ítem de a uno. El bucle hace lo mismo con los elementos de la lista, automáticamente.
    🎯
    Mini desafío: ¿Podés agregar una función reiniciar() conectada a un tercer botón que ponga puntos = 0, actualice el DOM, y vuelva a agregar "oculto" al ganador? ¡Intentalo vos solo antes de ver la solución!
    Referencia final
    El script.js completo
    Acá tenés todo el JavaScript junto, con comentarios explicando cada parte. Usalo como referencia si algo no te funciona.
    script.js — completo
    // ═══════════════════════════════════════════════════ // PASO 1: Variables de estado // ═══════════════════════════════════════════════════ let puntos = 0; // guarda el puntaje real (puede cambiar) const META = 10; // la meta para ganar (nunca cambia) // ═══════════════════════════════════════════════════ // PASO 2: Obtener elementos del DOM // ═══════════════════════════════════════════════════ // getElementById → busca 1 elemento por su id único const elNumero = document.getElementById("numero-puntos"); const elMensaje = document.getElementById("mensaje"); const elGanador = document.getElementById("ganador"); // getElementsByClassName → devuelve una LISTA de elementos const jugadores = document.getElementsByClassName("jugador"); // ═══════════════════════════════════════════════════ // PASO 3: Funciones con lógica // ═══════════════════════════════════════════════════ function sumarPunto() { puntos = puntos + 1; // 1. actualizar variable elNumero.textContent = puntos; // 2. reflejar en el DOM if (puntos >= META) { // 3. lógica de victoria elMensaje.textContent = "🎉 ¡Ganaste!"; elGanador.classList.remove("oculto"); for (let i = 0; i < jugadores.length; i++) { jugadores[i].classList.add("resaltado"); } } else { elMensaje.textContent = "Vas por " + puntos + " puntos"; } } function restarPunto() { if (puntos > 0) { puntos = puntos - 1; elNumero.textContent = puntos; elMensaje.textContent = "Bajaste a " + puntos + " puntos"; if (puntos < META) { elGanador.classList.add("oculto"); } } else { elMensaje.textContent = "⚠️ ¡No podés bajar de 0!"; } } function reiniciar() { puntos = 0; elNumero.textContent = puntos; elMensaje.textContent = "¡Reiniciado! Empezá de nuevo 💪"; elGanador.classList.add("oculto"); for (let i = 0; i < jugadores.length; i++) { jugadores[i].classList.remove("resaltado"); } }

    ¡Lo construiste de cero! 🎉

    Aprendiste a conectar HTML, CSS y JavaScript. Obtuviste elementos del DOM por id y por clase, usaste variables de estado, lógica con if/else, y manipulaste clases CSS desde JS. Eso es el corazón del desarrollo web frontend.

    ★ Modo avanzado desbloqueado

    El contador no era el límite.

    Estos retos no tienen solución en la guía. Vas a necesitar buscar en MDN, Stack Overflow o preguntar al profe. Eso es exactamente lo que hace un programador real.

    Nivel 1 — Cómodo
    Mejorar la UX del contador
    • Agregá un contador de "intentos" que cuenta cuántas veces pulsaron Sumar o Restar en total y lo muestre en pantalla.
    • Cambiá el color del número según el valor: rojo si es 0, amarillo si está entre 1 y 5, verde si es 6 o más. Usá style.color o agrega/quitá clases.
    • Deshabilitá el botón Restar cuando puntos === 0 usando elemento.disabled = true. Volvé a habilitarlo cuando suba.
    • Hacé que el título de la página cambie con document.title para mostrar los puntos actuales.
    Nivel 2 — Desafiante
    Múltiples jugadores con puntajes individuales
    • Modificá el proyecto para que cada jugador tenga su propio botón de sumar y su propio puntaje. Usá un array o un objeto para guardar los puntajes: let puntajes = [0, 0, 0].
    • Mostrá el nombre del jugador que va ganando (el que tiene más puntos) en tiempo real. Si hay empate, mostrá "Empate".
    • Buscá en MDN cómo funciona Math.max() y indexOf() para encontrar al jugador con más puntos.
    • Agregá una animación CSS de pulso al jugador que acaba de sumar un punto usando classList.add() y luego classList.remove() con setTimeout().
    Nivel 3 — Difícil
    Historial y localStorage
    • Creá un historial de movimientos: cada vez que se suma o resta, agregá una entrada a una lista visible en pantalla con el texto "Sumó 1 → ahora tiene X puntos".
    • Investigá cómo funciona document.createElement() y element.appendChild() para agregar elementos al DOM dinámicamente (sin tocar el HTML).
    • Guardá el puntaje en localStorage para que se mantenga aunque el usuario recargue la página. Investigá localStorage.setItem() y localStorage.getItem().
    • Agregá un botón "Deshacer" que revierta el último movimiento. Usá un stack (un array donde solo usás push() y pop()) para guardar el historial.
    Nivel 4 — Experto
    Refactorizar con addEventListener y querySelector
    • Eliminá todos los onclick="" del HTML. En su lugar, usá document.querySelector() para seleccionar los botones por clase o atributo, y conectalos con addEventListener("click", función). Esa es la forma profesional de hacerlo.
    • Reescribí el código usando querySelectorAll() en lugar de getElementsByClassName() y recorrelo con .forEach() en lugar de un bucle for.
    • Convertí toda la lógica del contador en un objeto con propiedades y métodos: const contador = { puntos: 0, sumar() {...}, restar() {...} }.
    • Investigá cómo funciona el evento keydown y hacé que las teclas ArrowUp y ArrowDown del teclado también sumen y resten puntos.
