¿Qué es una API?

Definición simple

Una API (Application Programming Interface) es un conjunto de reglas que permite a dos programas comunicarse entre sí. Piensa en ella como un mesero en un restaurante:

  • Tú (cliente): Tienes hambre y quieres comida
  • El mesero (API): Toma tu pedido y lo pasa a la cocina
  • La cocina (servidor): Prepara tu comida
  • El mesero (API): Te trae tu comida

¿Por qué necesitamos APIs?

  • Reutilización: No necesitas construir todo desde cero
  • Integración: Tu app puede usar servicios de otros
  • Escalabilidad: El backend procesa datos complejos
  • Seguridad: Los datos sensibles nunca llegan al navegador

Ejemplo cotidiano: Google Maps

Cuando usas Google Maps en un sitio web:

Tu navegador hace una solicitud a los servidores de Google: "Dame la ubicación de pizzerías en Bogotá" ↓ Los servidores de Google buscan en su base de datos ↓ Google responde con datos: [ {nombre: "Pizza Palace", lat: 4.7110, lng: -74.0055}, ... ] ↓ Tu navegador dibuja el mapa con esos datos
Idea clave: Una API es simplemente una forma estandarizada de pedir información a un servidor remoto.

Arquitectura REST

¿Qué es REST?

REST (Representational State Transfer) es un estilo de arquitectura para diseñar APIs web. Es el más usado en internet hoy en día.

Principios de REST

1. Cliente-Servidor

El cliente (navegador) y el servidor (donde viven los datos) están separados.

[Cliente: Tu navegador] ←→ [Servidor: Donde viven los datos]

2. Stateless (Sin estado)

Cada solicitud es independiente. El servidor no recuerda las solicitudes previas.

3. Recursos

Todo en REST es un recurso. Algunos ejemplos:

  • /usuarios - Recurso de usuarios
  • /productos - Recurso de productos
  • /publicaciones - Recurso de publicaciones

4. Operaciones estándar

En REST, usamos métodos HTTP para indicar qué queremos hacer:

  • GET: Obtener información
  • POST: Crear algo nuevo
  • PUT: Actualizar algo existente
  • DELETE: Eliminar algo

Ejemplo REST: Sistema de blog

GET /posts → Obtener todos los posts POST /posts → Crear un nuevo post GET /posts/5 → Obtener el post con ID 5 PUT /posts/5 → Actualizar el post con ID 5 DELETE /posts/5 → Eliminar el post con ID 5 GET /posts/5/comentarios → Obtener comentarios del post 5 POST /posts/5/comentarios → Crear comentario en el post 5
Patrón clave: REST es predecible. Solo viendo el verbo HTTP y la URL, ya sabes qué hace.

Respuestas REST

Los servidores REST responden generalmente con JSON (JavaScript Object Notation):

{ "id": 1, "nombre": "Juan", "email": "juan@example.com", "edad": 28 }

O un array de objetos:

[ { "id": 1, "nombre": "Juan" }, { "id": 2, "nombre": "María" }, { "id": 3, "nombre": "Carlos" } ]

Endpoints: La puerta de entrada a los datos

¿Qué es un endpoint?

Un endpoint es una URL específica en un servidor API que te permite acceder a un recurso o realizar una acción.

Estructura de un endpoint

https://api.ejemplo.com/v1/usuarios/42 └──────┬────────────┘ └┬┘ └──┬──┘ └─┬─┘ Protocolo Versión Recurso ID

Desglose:

  • https:// - Protocolo seguro (siempre usar en producción)
  • api.ejemplo.com - Dominio del servidor API
  • /v1 - Versión de la API (permite cambios sin romper código antiguo)
  • /usuarios - Recurso (qué tipo de datos quieres)
  • /42 - Identificador específico (usuario con ID 42)

Ejemplos de endpoints reales

GET Obtener todos los usuarios

https://api.ejemplo.com/usuarios

Retorna un array con todos los usuarios en la base de datos.

GET Obtener un usuario específico

https://api.ejemplo.com/usuarios/42

Retorna solo el usuario con ID 42.

POST Crear un nuevo usuario

https://api.ejemplo.com/usuarios

Envías datos del nuevo usuario, el servidor lo crea y retorna el usuario creado.

PUT Actualizar un usuario

https://api.ejemplo.com/usuarios/42

Envías datos actualizados del usuario 42, el servidor los actualiza.

DELETE Eliminar un usuario

https://api.ejemplo.com/usuarios/42

Elimina el usuario con ID 42 de la base de datos.

Parámetros en los endpoints

A veces necesitas enviar datos adicionales en la URL:

Query Parameters

Se usan para filtrar o paginar resultados:

https://api.ejemplo.com/usuarios?edad=30&ciudad=bogota&pagina=2 Parámetros: - edad=30 → Solo usuarios con edad 30 - ciudad=bogota → Solo usuarios de Bogotá - pagina=2 → Segunda página de resultados

Path Parameters

Se usan en la ruta para identificar recursos específicos:

https://api.ejemplo.com/usuarios/42/publicaciones/10 Parámetros: - 42 → ID del usuario - 10 → ID de la publicación
Diferencia clave: Query parameters (?) son opcionales y filtran. Path parameters (/) son obligatorios e identifican.

Métodos HTTP: El verbo de tu solicitud

¿Qué son los métodos HTTP?

Son como verbos que le dicen al servidor QUÉ quieres hacer con un recurso. Los 4 principales son:

GET - Obtener datos

Propósito: Traer información del servidor sin modificarla.

Seguro: Sí, no cambia nada en el servidor.

Idempotente: Sí, siempre retorna lo mismo.

Ejemplo GET

// Solicitud GET /usuarios HTTP/1.1 Host: api.ejemplo.com // Respuesta 200 OK [ { "id": 1, "nombre": "Juan" }, { "id": 2, "nombre": "María" } ]

POST - Crear datos

Propósito: Enviar datos al servidor para crear algo nuevo.

Seguro: No, cambia el estado del servidor.

Idempotente: No, cada solicitud crea un nuevo recurso.

Ejemplo POST

// Solicitud POST /usuarios HTTP/1.1 Host: api.ejemplo.com Content-Type: application/json { "nombre": "Pedro", "email": "pedro@example.com", "edad": 25 } // Respuesta 201 Created { "id": 3, "nombre": "Pedro", "email": "pedro@example.com", "edad": 25 }

PUT - Actualizar datos

Propósito: Reemplazar un recurso completo con nuevos datos.

Seguro: No, modifica datos existentes.

Idempotente: Sí, llamarlo múltiples veces produce el mismo resultado.

Ejemplo PUT

// Solicitud PUT /usuarios/3 HTTP/1.1 Host: api.ejemplo.com Content-Type: application/json { "nombre": "Pedro González", "email": "pedro.gonzalez@example.com", "edad": 26 } // Respuesta 200 OK { "id": 3, "nombre": "Pedro González", "email": "pedro.gonzalez@example.com", "edad": 26 }

DELETE - Eliminar datos

Propósito: Eliminar un recurso del servidor.

Seguro: No, elimina datos permanentemente.

Idempotente: Sí, borrar dos veces es lo mismo que borrar una.

Ejemplo DELETE

// Solicitud DELETE /usuarios/3 HTTP/1.1 Host: api.ejemplo.com // Respuesta 204 No Content (El servidor no retorna nada, solo confirma que fue eliminado)

Tabla comparativa

Método Propósito Tiene body Seguro Idempotente
GET Obtener datos No Sí Sí
POST Crear datos Sí No No
PUT Actualizar datos Sí No Sí
DELETE Eliminar datos No No Sí
Idempotente: Significa que llamar varias veces produce el mismo resultado que llamar una sola vez.

Fetch API: Haciendo solicitudes con JavaScript

¿Qué es Fetch?

fetch() es la forma moderna de hacer solicitudes HTTP desde JavaScript en el navegador. Reemplazó al antiguo XMLHttpRequest.

Sintaxis básica

fetch(url) .then(response => response.json()) .then(data => console.log(data)) .catch(error => console.error('Error:', error));

GET - Obtener datos

Ejemplo simple

fetch('https://api.ejemplo.com/usuarios') .then(response => response.json()) .then(usuarios => { console.log('Usuarios obtenidos:', usuarios); }) .catch(error => { console.error('Hubo un error:', error); });

Paso a paso

// 1. Hacer la solicitud (devuelve una Promise) const promise = fetch('https://api.ejemplo.com/usuarios'); // 2. Cuando la respuesta llega, procesarla promise.then(response => { // response es un objeto de respuesta HTTP console.log(response.status); // 200, 404, etc. console.log(response.ok); // true si status es 200-299 // 3. Convertir el body a JSON return response.json(); }) // 4. Ahora tenemos los datos reales .then(data => { console.log(data); // Array de usuarios }) // 5. Si algo sale mal .catch(error => { console.error('Error de red:', error); });

POST - Enviar datos

Crear un nuevo usuario

const nuevoUsuario = { nombre: 'Ana', email: 'ana@example.com', edad: 30 }; fetch('https://api.ejemplo.com/usuarios', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(nuevoUsuario) }) .then(response => response.json()) .then(usuarioCreado => { console.log('Usuario creado:', usuarioCreado); }) .catch(error => console.error('Error:', error));

Explicación de las opciones

Opción Significado Ejemplo
method Verbo HTTP 'POST', 'GET', 'PUT', 'DELETE'
headers Metadatos de la solicitud { 'Content-Type': 'application/json' }
body Datos a enviar JSON.stringify(objeto)

PUT - Actualizar datos

const usuarioActualizado = { nombre: 'Ana María', email: 'ana.maria@example.com', edad: 31 }; fetch('https://api.ejemplo.com/usuarios/5', { method: 'PUT', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(usuarioActualizado) }) .then(response => response.json()) .then(usuario => console.log('Usuario actualizado:', usuario)) .catch(error => console.error('Error:', error));

DELETE - Eliminar datos

fetch('https://api.ejemplo.com/usuarios/5', { method: 'DELETE' }) .then(response => { if (response.ok) { console.log('Usuario eliminado correctamente'); } }) .catch(error => console.error('Error:', error));

Manejando errores correctamente

fetch('https://api.ejemplo.com/usuarios') .then(response => { // Importante: fetch solo rechaza promesas en errores de RED // NO si el servidor responde con 404 o 500 if (!response.ok) { throw new Error(`Error HTTP: ${response.status}`); } return response.json(); }) .then(data => { console.log('Éxito:', data); }) .catch(error => { // Aquí llegan errores de red Y errores que lanzamos console.error('Error:', error.message); });
⚠️ Importante: fetch() NO rechaza la promesa con errores HTTP como 404 o 500. Solo rechaza en errores de red. Siempre verifica response.ok o response.status.

Async/Await: Código más limpio

¿Qué es async/await?

Es una forma más moderna y legible de trabajar con Promises. Permite escribir código asincrónico como si fuera sincrónico.

Sintaxis básica

async function obtenerUsuarios() { // El código aquí espera a que terminen las Promises const response = await fetch('https://api.ejemplo.com/usuarios'); const usuarios = await response.json(); return usuarios; } // Usar la función obtenerUsuarios() .then(usuarios => console.log(usuarios)) .catch(error => console.error(error));

Comparación: Promises vs Async/Await

Con .then() (Promises)

function obtenerDatos() { return fetch('/api/datos') .then(response => response.json() ) .then(data => { console.log(data); return data; }) .catch(error => console.error(error) ); }

Con async/await

async function obtenerDatos() { try { const response = await fetch('/api/datos'); const data = await response.json(); console.log(data); return data; } catch (error) { console.error(error); } }

¿Cuál prefieres leer? Probablemente la segunda, ¿verdad? Eso es async/await.

Las palabras clave

async

Transforma una función normal en una que retorna una Promise:

// Sin async: función normal function saludar() { return 'Hola'; } console.log(saludar()); // 'Hola' // Con async: retorna Promise async function saludar() { return 'Hola'; } console.log(saludar()); // Promise { 'Hola' } // Para obtener el valor: saludar().then(msg => console.log(msg)); // 'Hola'

await

Pausa la ejecución hasta que una Promise se resuelva. SOLO funciona dentro de funciones async:

async function ejemploAwait() { console.log('Antes'); const response = await fetch('/api/datos'); // Se pausa aquí hasta que fetch termine console.log('Después'); } // Esto NO funciona: // const data = await fetch(...); ❌ Error fuera de async

Manejo de errores con try/catch

En async/await, usamos try/catch en lugar de .catch():

async function obtenerUsuario(id) { try { // Código que queremos ejecutar const response = await fetch(`/api/usuarios/${id}`); if (!response.ok) { throw new Error(`Usuario no encontrado (${response.status})`); } const usuario = await response.json(); console.log('Usuario:', usuario); return usuario; } catch (error) { // Código si algo sale mal console.error('Error:', error.message); return null; // O hacer algo más } }

Ejemplo completo: CRUD de usuarios

async function obtenerUsuarios() { const response = await fetch('/api/usuarios'); return await response.json(); } async function obtenerUnUsuario(id) { const response = await fetch(`/api/usuarios/${id}`); if (!response.ok) throw new Error('Usuario no encontrado'); return await response.json(); } async function crearUsuario(datos) { const response = await fetch('/api/usuarios', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(datos) }); return await response.json(); } async function actualizarUsuario(id, datos) { const response = await fetch(`/api/usuarios/${id}`, { method: 'PUT', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(datos) }); return await response.json(); } async function eliminarUsuario(id) { const response = await fetch(`/api/usuarios/${id}`, { method: 'DELETE' }); if (response.ok) { console.log('Usuario eliminado'); } } // Usar las funciones async function main() { try { // Obtener todos const usuarios = await obtenerUsuarios(); console.log('Todos los usuarios:', usuarios); // Obtener uno const usuario = await obtenerUnUsuario(1); console.log('Usuario 1:', usuario); // Crear const nuevo = await crearUsuario({ nombre: 'Roberto', email: 'roberto@example.com' }); console.log('Usuario creado:', nuevo); // Actualizar const actualizado = await actualizarUsuario(nuevo.id, { nombre: 'Roberto García' }); console.log('Usuario actualizado:', actualizado); // Eliminar await eliminarUsuario(nuevo.id); } catch (error) { console.error('Error en main:', error); } } // Ejecutar main();
Consejo: async/await hace el código más fácil de leer y mantener. Úsalo en lugar de .then() cuando sea posible.

Ejemplo 1: Mostrar datos en una página

Obtener un list de usuarios y mostrarlos en HTML:

async function cargarUsuarios() { try { const response = await fetch('http://localhost:3000/usuarios'); const usuarios = await response.json(); // Obtener elemento HTML const listado = document.getElementById('lista-usuarios'); listado.innerHTML = ''; // Limpiar // Crear HTML para cada usuario usuarios.forEach(usuario => { const div = document.createElement('div'); div.innerHTML = `

${usuario.nombre}

Email: ${usuario.email}

Edad: ${usuario.edad}

`; listado.appendChild(div); }); } catch (error) { console.error('Error:', error); } } // Ejecutar cuando carga la página document.addEventListener('DOMContentLoaded', cargarUsuarios);

Ejemplo 2: Formulario para crear usuario

const form = document.getElementById('form-usuario'); form.addEventListener('submit', async (evento) => { evento.preventDefault(); // No recarga la página // Obtener valores del formulario const nombre = document.getElementById('nombre').value; const email = document.getElementById('email').value; const edad = document.getElementById('edad').value; try { const response = await fetch('http://localhost:3000/usuarios', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ nombre, email, edad }) }); if (response.ok) { const usuarioCreado = await response.json(); console.log('Usuario creado:', usuarioCreado); form.reset(); // Limpiar el formulario alert('Usuario creado exitosamente'); } else { alert('Error al crear usuario'); } } catch (error) { console.error('Error:', error); alert('Error de red'); } });

async function buscarUsuario(id) { try { const response = await fetch( `http://localhost:3000/usuarios/${id}` ); if (!response.ok) { throw new Error('Usuario no encontrado'); } const usuario = await response.json(); return usuario; } catch (error) { console.error('Error:', error); return null; } } // Usar const usuarioBuscado = await buscarUsuario(1); if (usuarioBuscado) { console.log('Encontrado:', usuarioBuscado); }

Ejemplo 4: Actualizar usuario

async function actualizarUsuario(id, nuevosDatos) { try { const response = await fetch( `http://localhost:3000/usuarios/${id}`, { method: 'PUT', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(nuevosDatos) } ); if (!response.ok) { throw new Error('Error al actualizar'); } const usuarioActualizado = await response.json(); console.log('Actualizado:', usuarioActualizado); return usuarioActualizado; } catch (error) { console.error('Error:', error); } } // Usar await actualizarUsuario(1, { nombre: 'Juan Pérez', edad: 35 });

Ejemplo 5: Borrar usuario con confirmación

async function eliminarUsuario(id) { // Pedir confirmación if (!confirm('¿Estás seguro de que quieres eliminar?')) { return; } try { const response = await fetch( `http://localhost:3000/usuarios/${id}`, { method: 'DELETE' } ); if (response.ok) { console.log('Usuario eliminado'); // Recargar la lista await cargarUsuarios(); } } catch (error) { console.error('Error:', error); } }

Crear tu propia API en localhost

¿Por qué tu propia API?

Para practicar, necesitas un servidor que responda. Aquí te mostraré cómo crear uno sencillo con Node.js y Express.

Requisitos

  • Node.js instalado (descargalo de nodejs.org)
  • Un editor de código
  • Terminal/Command Prompt

Paso 1: Crear la carpeta del proyecto

mkdir mi-api cd mi-api

Paso 2: Inicializar Node.js

npm init -y

Esto crea un archivo package.json.

Paso 3: Instalar Express

npm install express cors

cors permite que tu navegador se comunique con la API.

Paso 4: Crear el servidor

Crea un archivo llamado server.js:

const express = require('express'); const cors = require('cors'); const app = express(); const PORT = 3000; // Middleware app.use(cors()); // Permite solicitudes del navegador app.use(express.json()); // Parsea JSON // Base de datos simulada (en memoria) let usuarios = [ { id: 1, nombre: 'Juan', email: 'juan@example.com', edad: 28 }, { id: 2, nombre: 'María', email: 'maria@example.com', edad: 32 }, { id: 3, nombre: 'Carlos', email: 'carlos@example.com', edad: 25 } ]; let nextId = 4; // GET - Obtener todos los usuarios app.get('/usuarios', (req, res) => { res.json(usuarios); }); // GET - Obtener un usuario por ID app.get('/usuarios/:id', (req, res) => { const usuario = usuarios.find(u => u.id === parseInt(req.params.id)); if (!usuario) { return res.status(404).json({ error: 'Usuario no encontrado' }); } res.json(usuario); }); // POST - Crear un nuevo usuario app.post('/usuarios', (req, res) => { const { nombre, email, edad } = req.body; // Validar que vienen los datos if (!nombre || !email || !edad) { return res.status(400).json({ error: 'Faltan datos' }); } const nuevoUsuario = { id: nextId++, nombre, email, edad }; usuarios.push(nuevoUsuario); res.status(201).json(nuevoUsuario); }); // PUT - Actualizar un usuario app.put('/usuarios/:id', (req, res) => { const usuario = usuarios.find(u => u.id === parseInt(req.params.id)); if (!usuario) { return res.status(404).json({ error: 'Usuario no encontrado' }); } // Actualizar solo los campos que vienen if (req.body.nombre) usuario.nombre = req.body.nombre; if (req.body.email) usuario.email = req.body.email; if (req.body.edad) usuario.edad = req.body.edad; res.json(usuario); }); // DELETE - Eliminar un usuario app.delete('/usuarios/:id', (req, res) => { const index = usuarios.findIndex(u => u.id === parseInt(req.params.id)); if (index === -1) { return res.status(404).json({ error: 'Usuario no encontrado' }); } const usuarioEliminado = usuarios.splice(index, 1); res.json(usuarioEliminado[0]); }); // Iniciar el servidor app.listen(PORT, () => { console.log(`Servidor corriendo en http://localhost:${PORT}`); console.log(`Prueba: GET http://localhost:${PORT}/usuarios`); });

Paso 5: Ejecutar el servidor

node server.js

Deberías ver:

Servidor corriendo en http://localhost:3000
Prueba: GET http://localhost:3000/usuarios

Paso 6: Probar la API

Abre tu navegador y ve a:

http://localhost:3000/usuarios

Deberías ver el JSON con los usuarios.

Herramientas útiles para desarrolladores

Postman - Tester de APIs

Descarga: postman.com

Postman es una herramienta gráfica para probar APIs sin escribir código.

Ventajas:

  • Interfaz visual, no necesitas código
  • Guarda tus solicitudes para reutilizarlas
  • Organiza solicitudes en colecciones
  • Prueba todos los métodos HTTP fácilmente

Uso básico:

1. Abre Postman 2. Clic en "New Request" 3. Elige el método (GET, POST, etc.) 4. Coloca la URL: http://localhost:3000/usuarios 5. Si es POST, ve a "Body" y coloca JSON 6. Clic en "Send" 7. Ves la respuesta abajo

Insomnia - Alternativa moderna

Descarga: insomnia.rest

Similar a Postman pero más moderna y ligera.

F12 → Console:

  • Ejecuta JavaScript directamente
  • Ve logs de tus fetch()
  • Prueba fetch() en tiempo real

Curl - Desde la terminal

Si no quieres usar Postman, puedes usar curl:

# GET curl http://localhost:3000/usuarios # POST curl -X POST http://localhost:3000/usuarios \ -H "Content-Type: application/json" \ -d '{"nombre":"Ana","email":"ana@example.com","edad":28}' # PUT curl -X PUT http://localhost:3000/usuarios/1 \ -H "Content-Type: application/json" \ -d '{"nombre":"Juan García"}' # DELETE curl -X DELETE http://localhost:3000/usuarios/1

Extensiones útiles para Chrome

  • REST Client: Haz requests REST directamente
  • Talend API Tester: Similar a Postman
  • JSON Viewer: Visualiza JSON mejor

Ejercicios prácticos

Ejercicio 1: GET simple

Obtén la lista de usuarios de tu API local

Usa fetch() para llamar a: http://localhost:3000/usuarios

Ejercicio 2: POST - Crear usuario

Crea un nuevo usuario en la API

Usa POST para crear un usuario con nombre "TuNombre", email "tu@email.com" y edad 25

Ejercicio 3: PUT - Actualizar

Actualiza el usuario con ID 1

Cambia su nombre a "Juan García" usando PUT

Ejercicio 4: DELETE - Eliminar

Elimina el usuario con ID 3

Usa DELETE. ¿Qué código de estado esperas?

Ejercicio 5: Manejo de errores

Obtén un usuario que NO existe (ID 999)

Intenta obtener el usuario 999 y maneja el error apropiadamente

Desafío final

Crea una página HTML que:
  1. Muestre la lista de todos los usuarios (GET /usuarios)
  2. Tenga un formulario para crear usuarios (POST /usuarios)
  3. Pueda actualizar usuarios existentes (PUT /usuarios/:id)
  4. Pueda eliminar usuarios (DELETE /usuarios/:id)
  5. Maneje todos los errores apropiadamente
